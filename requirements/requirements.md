# home - 要件定義（v2）

## 背景
家庭の雑務（送迎調整、スケジュール管理、TODO、買い物等）が多く、手作業で管理するのが面倒。
夫婦2人で共有でき、日々の面倒ごとをサポートしてくれるツールが欲しい。

## 対象ユーザー
- オーナー（夫・とっちゃん）— Android
- 妻（かあか）— iPhone
- 主にスマホから利用（PWA）

---

## 技術スタック
- Next.js 16 + React 19 + TypeScript
- Tailwind CSS 4
- Supabase（DB + 認証なし・RLSオープン）
- Vercel（デプロイ）
- PWA（Service Worker）

## データベース（Supabase）

### pickups テーブル
送迎の担当管理。date + type でユニーク制約。
| カラム | 型 | 説明 |
|---|---|---|
| id | uuid | PK |
| date | date | 日付 |
| type | text | "送り" or "迎え" |
| assignee | text | "とっちゃん" or "かあか" or null |

### events テーブル
家族・個人の予定。
| カラム | 型 | 説明 |
|---|---|---|
| id | uuid | PK |
| date | date | 日付 |
| title | text | 予定内容 |
| start_time | time | 開始時刻（任意） |
| end_time | time | 終了時刻（任意） |
| category | text | "家族" or "川村" or "萌香" |

### todos テーブル
| カラム | 型 | 説明 |
|---|---|---|
| id | uuid | PK |
| title | text | やること |
| date | date | 期限日（任意） |
| url | text | 関連URL（任意） |
| done | boolean | 完了フラグ |

### shopping_items テーブル
| カラム | 型 | 説明 |
|---|---|---|
| id | uuid | PK |
| name | text | 品名 |
| done | boolean | 購入済みフラグ |

## 機能一覧

### Phase 1（実装済み）

#### 1. カレンダー（月表示）— デフォルトタブ
- 月間カレンダーグリッド（TimeTreeライク）
- 日付タップで詳細モーダル表示
- 送迎: 1行で左=送り、右=迎え。担当者の頭文字+色で表示
- ゴミの日: 燃/缶/PETラベル
- 在宅勤務: 「在宅」ラベル
- 祝日: 祝日名表示、送迎スロット非表示
- 横スワイプで前月/翌月ナビゲーション

#### 2. カレンダー（週表示）— 週タブ
- 横1行×7行のリスト形式
- 送迎担当ボタンを各日に表示
- タグ行: 祝日/在宅/ゴミの日
- 予定も表示（家族=赤、川村=青、萌香=ピンク）
- 日付タップで詳細モーダル
- 横スワイプで前週/翌週ナビゲーション

#### 3. 日付詳細モーダル
- 送迎の担当選択（とっちゃん/かあかのトグルボタン）
- 既存の予定一覧
- 「+ 予定」ボタンで予定追加（カテゴリ選択: 家族/川村/萌香）
- 「+ TODO」ボタンでTODO追加（日付が自動セット）
- 入力時にスクロールが上に飛ばない（scrollIntoView: center）

#### 4. 送迎管理
- 平日（月〜金・祝日除外）に「送り」「迎え」スロットを自動生成
- 担当: とっちゃん / かあか
- 楽観的UI更新（選択即反映、リロードなし）
- バックグラウンドでSupabaseに書き込み
- 未調整日数をバナーで表示

#### 5. ゴミの日
- 火曜・金曜: 燃えるゴミ
- 第1・3・5木曜: カンビン
- 第2・4木曜: ペットボトル
- 自動表示（手動登録不要）

#### 6. 保育園休み
- 土日+祝日は保育園休み → 送迎スロット非表示
- 2026年の日本の祝日データ内蔵

#### 7. TODO
- TODO一覧タブ
- 日付（期限）付きTODO
- URL添付可能
- 完了/未完了トグル
- 完了済みは折りたたみ表示
- 月/週の日付詳細からもTODO追加可能

#### 8. 買い物リスト
- 買い物リストタブ
- 品名入力で即追加
- 購入済みチェック
- 購入済みの一括削除

#### 9. ダークモード
- prefers-color-scheme による自動切替
- CSS変数ベースのテーマ

#### 10. PWA
- Service Worker（オフラインキャッシュ）
- マニフェスト設定
- ホーム画面追加対応

### Phase 2以降（バックログ）
- お金の精算（立替清算）
- 支払い管理（PayPay/楽天Pay/クレカ）
- 毎月の支出負担管理
- クレジット明細からの光熱費抽出
- 資産管理
- プッシュ通知（リマインド）
- カレンダーアプリ連携（ical等）
