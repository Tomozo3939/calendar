# home - 子会社 CLAUDE.md

> 親会社ルール: @../parent-company/CLAUDE.md を参照

---

## プロジェクト概要

- **事業内容**: 家庭の雑務を一元管理・自動化するアプリ開発
- **目標**: 夫婦の家事・家計・スケジュールを楽にするツールを作る
- **期限**: 2026年4月中に第1機能リリース
- **対象ユーザー**: オーナーと妻（2人利用）
- **収益モデル**: なし（自家用ツール）

---

## 使用エージェント

| エージェント | 役割 |
|---|---|
| home--ceo | 事業方針・優先度決定 |
| home--pm | タスク管理・開発スプリント管理 |
| home--secretary | 日報・TODO管理 |
| home--review | 品質レビュー（CEO提出前必須） |

---

## 機能スコープ

### Phase 1（4月中リリース）
優先度の高い機能から1つ選定してリリースする。候補:
- 買い出しリスト（妻との共有）
- 月次精算（誰が何を立替えたかの清算）
- 支払い管理（PayPay/楽天Pay/クレカの清算追跡）

### Phase 2以降
- カレンダー連携（TimeTree）
- 日程調整
- 資産管理
- TODO管理
- リマインド通知
- 毎月の支出項目・負担者管理
- クレジット明細からの光熱費抽出・表への落とし込み

---

## 技術方針

- **フレームワーク**: Next.js（既存PJのcrypto-dashboardと技術スタック統一）
- **デプロイ**: Vercel
- **認証**: 2人利用のため簡易認証（パスワードまたはLINE連携）
- **データ**: Supabase or ローカルJSON（規模に応じて判断）
- **モバイル対応**: PWA（スマホからの利用が主）

## ディレクトリ構成

```
home/
├── reference/      ... 参考資料
├── output/         ... 成果物（ここにのみ最終アウトプットを保管）
├── report/         ... 作業報告（日報）
├── ideas/          ... アイデアメモ
├── scripts/        ... スクリプト
├── drafts/         ... 下書き
├── analytics/      ... 分析データ
├── config/         ... プロジェクト固有設定
└── requirements/   ... 要件定義
```

---

## ルール

- 成果物は必ず `output/` に保管する（agentフォルダに入れない）
- 日報は毎日 `report/YYYY-MM-DD.md` に作成する（改善提案必須）
- レビューagentを通してからCEOに提出する
- AIっぽいアウトプットは禁止（ui-design-guidelinesスキル参照）
- 妻も使うため、UIは極力シンプルに。説明不要で使えることを最優先にする
